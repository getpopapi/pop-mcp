MCP Server

POP MCP Server overview

POP MCP exposes the POP Cloud API as a remote, stateless Model Context Protocol endpoint for AI clients and developer tooling.

What it is

The server gives assistants and automation tools a direct way to use POP capabilities without embedding business logic into the client. The public landing page is human-readable, while /mcp is the protocol endpoint.

Core routes

  • / for the marketing and documentation landing page
  • /mcp for Streamable HTTP MCP traffic
  • /health for a lightweight JSON health check

Transport and auth

  • Transport: Streamable HTTP MCP
  • Request headers: Content-Type: application/json and Accept: application/json, text/event-stream
  • Authentication: Authorization: Bearer <license_key>
  • Model: stateless and multi-tenant

At a glance

Best first check

Call tools/list to confirm the server is reachable and the tool surface is exposed.

Primary audience

Developers, AI agents, and integration tools that need POP document workflows.

Related pages

Connect, Tools, and Onboarding.

Verification flow

Use this sequence to confirm the live endpoint works:

  1. Open the homepage or this overview page.
  2. Inspect the /mcp endpoint with your POP license key.
  3. Run tools/list to verify the server is advertising the available tools.
  4. Use the homepage tester or MCP Inspector to validate a real request end to end.
