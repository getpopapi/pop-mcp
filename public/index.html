How to connect

Point your MCP client to https://mcp.popapi.io/mcp and send your POP license key as a Bearer token. The server is stateless and multi-tenant: every request carries its own credential.

Keep the header on every request

Quick test

The first real protocol check is tools/list. It confirms request routing, auth handling, MCP negotiation, and tool exposure in one call. 

curl -X POST https://mcp.popapi.io/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "Authorization: Bearer your_license_key_here" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
Open /health

What success looks like

200 OK on initialize

Confirms the transport layer and request lifecycle are working.

200 OK on tools/list

Confirms the server is advertising its MCP tool surface correctly.

401 unauthorized_user without Bearer token

Confirms auth enforcement is behaving correctly on the live endpoint.

Production by default, with optional request-level environment switches for supported tools.

Protocol profile

Streamable HTTP MCP. Keep Accept: application/json, text/event-stream on POST requests.

Public root path

Intentionally human-readable. The MCP protocol endpoint remains isolated on /mcp.
